# Next.js プロジェクトレビュー依頼
以下を理解した上で、技術的観点に特化したコードレビューをお願いします。

---

## 1. レビュー目的

このプロジェクトは **Next.js 初学者が学習目的で作成した練習用プロジェクト**です。
実装が Next.js / TypeScript / Prisma / Zod / TailwindCSS のベストプラクティスに沿っているかを確認し、改善点を明確にしたいと考えています。

ユーザビリティやアプリとしての完成度は主目的ではありません。
あくまで **実装品質・設計レベルでの理解深化が目的**です。

---

## 2. プロジェクト概要

購入品の登録・更新ができるシンプルな CRUD アプリケーションです。（削除はありません。）

---

## 3. 使用技術（バージョン含む）

* Next.js 16（App Router）
* React 19
* TypeScript
* Prisma（PostgreSQL）
* Zod
* Tailwind CSS
* Biome
* パッケージマネージャー：pnpm

その他細かなライブラリはコードを直接参照してください。

---

## 4. 学習対象として意識している範囲

レビューの前提として、以下の技術要素を特に学びたいと考えています。

* Next.js App Router における
  * Server Component / Client Component の責務分離
  * Server Action（useActionState）の正しい利用方法
* Prisma を使った型安全な DB アクセス
* Zod を使ったバリデーション設計と分離
* TailwindCSS による最低限のスタイリング（デザイン品質は範囲外）
* ディレクトリ構成、コンポーネント構造


---

## 5. レビュー観点（基本）

* 各技術スタックのベストプラクティスに沿っているか
* 冗長ロジックが存在しないか
* Server Component / Client Component の分離は妥当か
* サーバーアクション使用方法が適切か
* Prisma の扱いが型安全で適切か
* Zod バリデーションの設計が適切か

---

## 6. レビュー観点（応用）

* 拡張性・保守性
* セキュリティ
* 可読性
* エラーハンドリング
* ディレクトリ構成および責務分割の妥当性
* キャッシュ戦略やデータフェッチパターン（必要な場合）
* その他指摘すべきと判断したことがあれば

---

## 7. 特に重点的に見てほしいポイント

* コンポーネント構造とファイル構成が Next.js 16 の推奨に沿っているか
* サーバーアクションの扱い方について改善余地があるか
* Prisma + Zod の型安全な統合方法は適切か
* リファクタリング可能なポイント
* 「次にステップアップするための改善案」があれば提示してほしい

---

## 8. レビュー時のトーン・深さ・出力形式

以下の形式でレビューをお願いします。

1. 指摘内容
2. その理由（根拠・Next.js の推奨事項があれば出典も歓迎）
3. 修正例（TypeScript + pnpm 前提で具体的なコード）
4. 重大度（High / Medium / Low）

レビューは「厳しめ」で構いません。
実務で通じるスキルが欲しいので技術的な観点では本格的にレビューを行なってください。

---

## 9. 非レビュー対象（触れなくてよい部分）

* デザイン性・UI/UX（Tailwind の細かいデザイン指摘は不要）
* 本番運用を想定した高度なアーキテクチャ提案
* DB スキーマの大規模リデザイン
* 複雑なビジネスロジックの想定

---

## 10. 補足
開発環境の立ち上げは以下のコマンドを利用する
```shell
pnpm dev
```
